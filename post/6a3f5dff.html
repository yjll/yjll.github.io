DataX同步全量Mqsql数据到Elasticsearch

DataX 是阿里巴巴集团内被广泛使用的离线数据同步工具/平台，实现包括 MySQL、SQL Server、Oracle、PostgreSQL、HDFS、Hive、HBase、OTS、ODPS 等各种异构数据源之间高效的数据同步功能。

DataX本身作为数据同步框架，将不同数据源的同步抽象为从源头数据源读取数据的Reader插件，以及向目标端写入数据的Writer插件，理论上DataX框架可以支持任意数据源类型的数据同步工作。同时DataX插件体系作为一套生态系统, 每接入一套新数据源该新加入的数据源即可实现和现有的数据源互通。

编译

官方提供的二进制包缺少必要的组件，我们需要下载源码自己编译

wget https://codeload.github.com/alibaba/DataX/zip/master

解压后编辑pom.xml，只保留部分必要module和mysql，es即可

<modules>
    <module>common</module>
    <module>core</module>
    <module>transformer</module>

    <!-- reader -->
    <module>mysqlreader</module>

    <!-- writer -->
    <module>elasticsearchwriter</module>

    <!-- common support module -->
    <module>plugin-rdbms-util</module>
    <module>plugin-unstructured-storage-util</module>
    <module>hbase20xsqlreader</module>
    <module>hbase20xsqlwriter</module>
    <module>kuduwriter</module>
</modules>

更改配置后打包


mvn -U clean package assembly:assembly -Dmaven.test.skip=true

ls ./target/datax/datax/
bin		conf		job		lib		log		log_perf	plugin		script		tmp

设置动态模板

开始同步数据前，在es中创建一个动态模板

PUT _template/hkey_transferbill
{
    "index_patterns":"hkey_transferbill",
    "settings":{
        "number_of_shards":2,
        "number_of_replicas":1
    },
    "mappings":{
        "data":{
            "dynamic_templates":[
                {
                    "keyword_field":{
                        "unmatch":"*Name",
                        "match_mapping_type":"string",
                        "mapping":{
                            "type":"keyword"
                        }
                    }
                },
                {
                    "text_fields":{
                        "match":"*Name",
                        "match_mapping_type":"string",
                        "mapping":{
                            "type":"text",
                            "fields":{
                                "raw":{
                                    "type":"keyword"
                                }
                            }
                        }
                    }
                }
            ],
            "properties":{
                "rowkey":{
                    "type":"keyword"
                },
                "age":{
                    "type":"keyword"
                }
            }
        }
    }
}

编写同步配置文件

MysqlReader 插件文档

ElasticSearchWriter 插件文档

同步时间类型问题，部分mysql中时间字段为空，导入到ES中时间变成了系统时间，我在导入时将ES中column的type更改为text解决了这个问题。

setting.speed.channel为并发数，setting.speed.byte是最大同步字节数

{
    "job":{
        "setting":{
            "speed":{
                "channel":4,
                "byte": 104857600
            }
        },
        "content":[
            {
                "reader":{
                    "name":"mysqlreader",
                    "parameter":{
                        "username":"root",
                        "password":"test",
                        "column":[
                            "code",
                            "internal_code",
                            "internal_code_md5"
                        ],
                        "connection":[
                            {
                                "jdbcUrl":[
                                    "jdbc:mysql://192.168.1.200:3306/test"
                                ],
                                "table":[
                                    "code_t"
                                ]
                            }
                        ]
                    }
                },
                "writer":{
                    "name":"elasticsearchwriter",
                    "parameter":{
                        "endpoint":"http://192.168.1.200:9200",
                        "accessId":"1",
                        "accessKey":"1",
                        "index":"code_t",
                        "type":"_doc",
                        "cleanup":true,
                        "dynamic":true,
                        "batchSize":5000,
                        "splitter":",",
                        "column":[
                            {
                                "name":"code",
                                "type":"long"
                            },
                            {
                                "name":"internal_code",
                                "type":"text"
                            },
                            {
                                "name":"internal_code_md5",
                                "type":"text"
                            }
                        ]
                    }
                }
            }
        ]
    }
}
